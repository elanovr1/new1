# 智能电销王 Android APP - 项目概览

## 📱 项目简介

**智能电销王**是一款基于Odoo客户管理系统的专业电销Android应用，通过自动拨号、智能队列管理、通话记录等功能，大幅提升电销团队的工作效率。

---

## 🎯 核心价值

### 对销售人员
- ⏱️ **节省时间**: 自动拨号减少90%的手动操作时间
- 📈 **提高效率**: 每小时可拨打50-100个客户（传统方式约20-30个）
- 📝 **规范管理**: 强制跟进记录，避免客户流失
- 📊 **数据透明**: 实时查看拨打统计和业绩排名

### 对管理者
- 👀 **透明监控**: 实时查看团队拨打情况
- 📊 **数据分析**: 全面的统计报表和趋势分析
- 🎯 **目标管理**: 设定目标并追踪完成情况
- 🔄 **系统集成**: 与Odoo后台无缝对接

## ✨ 功能特性

### 1. 自动拨号系统
```
核心功能：
├── 智能队列管理
│   ├── 按优先级排序（A级客户优先）
│   ├── 按时间排序（很久没联系的优先）
│   └── 随机拨打
├── 拨号间隔控制
│   ├── 5秒间隔
│   ├── 10秒间隔
│   ├── 30秒间隔
│   └── 手动控制
├── 实时控制
│   ├── 暂停/继续
│   ├── 跳过当前
│   └── 停止拨号
└── 拨号统计
    ├── 总拨打数
    ├── 成功接通数
    ├── 接通率
    └── 耗时统计
```

### 2. 客户管理
- 客户列表展示（与Odoo后台实时同步）
- 客户详情查看
- 全文搜索（姓名、手机、微信）
- 多维度筛选（类型、等级、状态）
- 离线缓存（无网络也可查看）

### 3. 跟进管理
- 通话后自动弹出跟进窗口
- 快速选择跟进结果
- 跟进模板（常用话术）
- 设置下次跟进时间
- 跟进历史查看
- 实时同步到Odoo

### 4. 统计报表
- 今日/本周/本月统计
- 拨打数量趋势图
- 接通率分析
- 团队排行榜
- 个人业绩追踪

## 🏗️ 技术架构

### 技术栈
```
前端框架: Android Native (Java/Kotlin)
最低版本: Android 6.0 (API 23)
目标版本: Android 13 (API 33)
架构模式: MVVM
```

### 核心技术
```
网络通信: Retrofit + OkHttp
数据库: Room (SQLite)
图片加载: Glide
依赖注入: 手动DI
异步处理: Handler + AsyncTask
```

### 后端对接
```
协议: JSON-RPC / XML-RPC
认证: Session Cookie
API: Odoo Standard API
同步: 增量同步 + 离线缓存
```

## 🚀 快速开始

### 1. 环境要求
- Android Studio Hedgehog (2023.1.1+)
- JDK 11+
- Android SDK (API 23-34)

### 2. 克隆项目
```bash
git clone [项目地址]
cd sales_dialer_app
```

### 3. 配置Odoo服务器
编辑 `app/src/main/java/com/salesdialer/config/ApiConfig.java`

### 4. 构建运行
```bash
# 清理项目
./gradlew clean

# 构建Debug版本
./gradlew assembleDebug

# 安装到设备
./gradlew installDebug
```

## 🎯 使用场景

### 场景1: 每日例行拨打
```
销售人员每天需要拨打100个客户

传统方式：
- 手动查找客户: 10秒/个
- 手动拨号: 5秒/个
- 等待拨通: 15秒/个
- 通话后记录: 30秒/个
总计: 60秒/个 × 100 = 6000秒 ≈ 100分钟

使用自动拨号：
- 自动查找拨号: 0秒
- 自动拨号: 5秒间隔
- 通话后快速跟进: 10秒/个
总计: 15秒/个 × 100 = 1500秒 ≈ 25分钟

节省时间: 75分钟 (75%)
```

### 场景2: A级客户优先跟进
```
需要优先联系50个A级客户

传统方式：
1. 筛选A级客户（手动）
2. 逐个拨打
3. 容易遗漏

使用自动拨号：
1. 选择"优先级优先"策略
2. 一键启动
3. 系统自动按A→B→C→D顺序拨打
4. 确保A级客户优先联系
```

### 场景3: 团队业绩竞赛
```
周一启动拨打竞赛，周五统计

传统方式：
- 人工统计各成员拨打数
- 容易出错
- 无法实时查看

使用智能电销王：
- 实时查看排行榜
- 自动统计拨打数
- 激励团队竞争
```

## 💡 最佳实践

### 拨号策略建议
```
早上 (9:00-11:00):
├── 策略: 优先级优先
├── 间隔: 10秒
└── 范围: A级客户

下午 (14:00-17:00):
├── 策略: 时间优先
├── 间隔: 5秒
└── 范围: B、C级客户

晚上 (19:00-21:00):
├── 策略: 随机拨打
├── 间隔: 30秒
└── 范围: 全部客户
```

### 跟进技巧
```
接通后：
1. 快速确认客户需求
2. 记录关键信息
3. 选择准确的跟进结果
4. 设置合理的下次跟进时间

未接通：
1. 选择"待跟进"
2. 记录拨打时间
3. 设置下次跟进时间（建议2小时后）

拒绝接听：
1. 选择"无意向"
2. 记录拒绝原因
3. 适当降低客户等级
```
